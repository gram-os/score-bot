# Raspberry Pi Deployment Guide

End-to-end guide for hosting score-bot on a Raspberry Pi with automatic deploys triggered by pushes to `main`.

---

## Hardware

| Component | Used |
|-----------|------|
| Board | Raspberry Pi 4 Model B — 4 GB RAM |
| Storage | 32 GB microSD card |
| Cooling | Heatsink case + fan |
| Network | Ethernet + Wi-Fi |

---

## 1. OS Setup

Flash **Raspberry Pi OS Lite (64-bit, Bookworm)** using [Raspberry Pi Imager](https://www.raspberrypi.com/software/). In the imager's advanced settings (gear icon):

- Enable SSH
- Set a username (e.g. `admin`) and password
- Set a hostname (e.g. `scorebot`)
- Configure Wi-Fi if needed

Boot the Pi and SSH in:

```bash
ssh admin@scorebot.local
```

Update the system:

```bash
sudo apt update && sudo apt upgrade -y
sudo apt install -y git curl ca-certificates make
```

---

## 2. Docker

```bash
curl -fsSL https://get.docker.com | sh
sudo usermod -aG docker $USER
```

Log out and back in, then verify:

```bash
docker run --rm hello-world
```

Docker Compose is bundled with modern Docker (`docker compose` — no separate install needed).

---

## 3. Clone and Configure the Bot

```bash
git clone https://github.com/gram-os/score-bot.git
cd score-bot
make setup          # copies .env.example → .env, generates SECRET_KEY
nano .env           # fill in all required values (see below)
```

Required `.env` values:

```
DISCORD_TOKEN=
DISCORD_CHANNEL_ID=
DISCORD_CLIENT_ID=
DISCORD_CLIENT_SECRET=
DISCORD_REDIRECT_URI=https://admin.yourdomain.com/auth/callback
ADMIN_DISCORD_IDS=
WEBHOOK_SECRET=        # auto-generated by make setup
REPO_PATH=/home/admin/score-bot
```

Start the services:

```bash
make up
make logs   # verify bot connects and web is listening on :8000
```

## 4. Boot Persistence

The Docker services are configured with `restart: unless-stopped` — they come back automatically after a reboot without any extra setup.

Verify the restart policy is active:

```bash
docker inspect score-bot-bot-1 --format '{{.HostConfig.RestartPolicy.Name}}'
# → unless-stopped
```

## 5. Cloudflare Tunnel

The tunnel exposes the bot's admin panel and the webhook listener to the internet without port forwarding. Requires a domain managed by Cloudflare.

### Install cloudflared

```bash
curl -L https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-arm64.deb -o cloudflared.deb
sudo dpkg -i cloudflared.deb
```

### Create the tunnel

```bash
cloudflared tunnel login        # opens a browser link — paste it on your machine
cloudflared tunnel create score-bot
cloudflared tunnel list         # note the tunnel UUID
```

### Route subdomains

```bash
cloudflared tunnel route dns score-bot deploy.gram-os.dev
cloudflared tunnel route dns score-bot admin.gram-os.dev
```

### Config file

```bash
nano ~/.cloudflared/config.yml
```

```yaml
tunnel: <tunnel-uuid>
credentials-file: /home/admin/.cloudflared/<tunnel-uuid>.json
ingress:
  - hostname: deploy.gram-os.dev
    service: http://localhost:9001
  - hostname: admin.gram-os.dev
    service: http://localhost:8000
  - service: http_status:404
```

### Install as a system service

```bash
sudo cloudflared --config /home/admin/.cloudflared/config.yml service install
sudo systemctl enable cloudflared
sudo systemctl start cloudflared
sudo systemctl status cloudflared   # should show active (running)
```

## 6. Webhook Listener

The webhook listener runs as a systemd service (outside Docker) so it can trigger `make restart` without killing itself.

### Install dependencies

```bash
python3 -m venv /home/admin/score-bot/deploy/.venv
/home/admin/score-bot/deploy/.venv/bin/pip install -r deploy/requirements.txt
```

### Install the systemd service

```bash
sudo cp deploy/webhook_listener.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable webhook_listener
sudo systemctl start webhook_listener
sudo systemctl status webhook_listener   # should show active (running)
```

## 7. GitHub Webhook

In the repo on GitHub: **Settings → Webhooks → Add webhook**

| Field | Value |
|-------|-------|
| Payload URL | `https://deploy.gram-os.dev/deploy` |
| Content type | `application/json` |
| Secret | value of `WEBHOOK_SECRET` from `.env` |
| Events | Just the push event |

GitHub will send a ping — a green checkmark confirms the tunnel and listener are reachable.

---

## 8. Auto-Deploy Flow

```
Push to main
     │
     ▼
GitHub webhook → POST https://deploy.gram-os.dev/deploy
     │
     ▼
webhook_listener.py verifies HMAC signature
     │
     ▼
git pull origin main && make restart
     │
     ▼
Docker images rebuilt, services restarted with new code
```
